* Added properties &quot;persistTo&quot; and &quot;replicateTo&quot; properties
  for the manager in order to configure the number of nodes
  to consider (new features of new couchbase client).
* The default transcoder has been moved to a factory level.
  Now the transcoder is assigned using a CouchbaseConnectionFactory.
* Added an empty class com.tangosol.net.CacheFactory in
  order to avoid ClassNotFoundException. Workaround until a
  proper fix for bug:
  http://java.net/jira/browse/GLASSFISH-18994
* Fixes some issues when shutting fown the manager. Not it
  works well.
